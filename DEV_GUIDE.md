## OpenCode Rules

- Respect the current delivery scope.
- Never implement future deliveries unless requested.
- Never execute Gradle, Maven or the application unless explicitly requested.
- Never create unnecessary files.
- Keep the architecture clean.
- Always use English for code, classes, methods and variables.

## Coding Standards

- JavaDoc on public classes.
- Organized code sections.
- Conventional Commits.
- Integer identifiers.
- Clean architecture.
- English naming convention.
- Consistent formatting.

## Commit Convention

Examples:

feat: implement persistence layer with JPA entities

feat: add domain models using MapStruct

feat: implement service layer

docs: configure Swagger documentation

fix: resolve entity relationship mapping
